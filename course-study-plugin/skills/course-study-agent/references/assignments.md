# Assignment Analysis and Planning

1. Read the assignment source and any rubric, template, dataset description, submission rules, and referenced lecture topics.
2. Extract the task objective, deliverables, constraints, grading signals, dependencies, deadlines when supplied, and ambiguous requirements.
3. Connect each deliverable to the relevant course knowledge.
4. Produce a staged completion plan with checkpoints and verification steps.
5. Offer pseudocode, outlines, examples, debugging help, or a clearly labelled draft when requested.

Do not fabricate experiments, results, citations, implementation status, personal reflection, or teacher feedback. Do not claim the student completed work that was not verified. Follow stricter academic-integrity rules from the project or institution when present.
