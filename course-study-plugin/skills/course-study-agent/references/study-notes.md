# Study Notes

Use the project's template when one exists. Otherwise use this minimal structure:

```markdown
# <Week/Chapter> - <Topic>

## Source
- <source path>

## 学习目标
- ...

## 核心知识点
### 1. <Concept>
- 中文讲解
- Key terms:
- Example / application:
- Common misunderstanding:

## 章节总结
- ...

## 待确认内容
- ...
```

Prefer the sequence explicitly shown in the source. If chapter/week or topic must be inferred, state the evidence to the user. Before creating a note, compare both sequence and topic with existing filenames and indexes; update an existing match instead of creating a duplicate.

Do not automatically add practice questions, a fixed question count, a bilingual answer format, or personal-review sections unless the project rules or user request requires them.
