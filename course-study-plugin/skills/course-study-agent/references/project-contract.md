# Project Contract

Use this reference when first applying the skill to a project.

## Discovery

1. Find the nearest applicable `AGENTS.md` and read it completely.
2. Inspect root filenames before opening course content.
3. Identify how the project represents courses, raw materials, notes, outputs, indexes, status, and logs.
4. Read only the target course or subject area unless cross-course work is requested.

## Rule Boundary

The Plugin owns reusable capabilities: file ingestion, study-note generation, assignment planning, exam review, mock-exam preparation, retrieval, discovery, fingerprinting, deduplication, and validation.

The project owns all local policy, including folder names, naming rules, note sections, languages beyond the Chinese default, question formats and counts, source registration, log format, update order, academic-integrity constraints, and cross-course boundaries.

If a project supplies no structure, report the proposed folders and filenames before making a large multi-course setup. For a single requested output, use a conservative location close to the source or ask when placement materially changes the result.

## Safety

- Never silently modify original course materials.
- Do not scan unrelated courses to answer a narrow question.
- Do not process archives or unknown formats without identifying their contents and intended scope.
- Preserve citations, page references, uncertainty, and teacher-specific emphasis when present.
- Keep personal course material local unless the user explicitly authorizes external sharing.
